## 1. Executive Summary

**LifeTrack** — минималистичный мобильный трекер ежедневных привычек с бинарной системой: делал или нет. Пользователь каждое утро за 5 секунд фиксирует вчерашний день — тап по карточке = ✓, не тронул = —. Визуальный прогресс в стиле GitHub contribution graph мотивирует не разрывать серию.

**Ключевая гипотеза:** Люди бросают habit-трекеры, потому что те требуют слишком много усилий. Если свести чек-ин к 5 тапам без принятия решений — retention будет выше, чем у конкурентов.

**Платформы:** iOS + Android (React Native + Expo)  
**Бюджет:** $0 (solo-разработка) + $119/год Apple Developer Account  
**Таймлайн MVP:** Proof of Work Challenge (30 дней)

### Эволюция дизайна

- **v1–v5:** Слайдер 0–10 → пользователи: «что значит сон 7 vs сон 8?»
- **v6–v7:** Слайдер 0–5 с текстовыми лейблами → друг: «выглядит как что-то, в чём нужно разобраться»
- **v8 (текущий):** Бинарный — тап по карточке ✓/—. Ноль когнитивной нагрузки

**Философия:** Цель — сформировать привычку, не измерить её. Успех = хоть какой-то прогресс. Сначала научись делать каждый день, потом уходи в детали.

## 2. Проблема и бизнес-цели

### 2.1. Проблема

Человек ставит цели — нормально питаться, заниматься спортом, работать над проектами. Без систематического отслеживания цели забываются через 1–2 недели. Существующие решения перегружены: КБЖУ, таймеры, шкалы, аналитика — и сами становятся источником стресса.

**Личная боль:** Жизненные обстоятельства стали мотивацией наладить 5 ключевых направлений. Ежедневный трекинг в ежедневнике дал результат за год — но инструмента, который был бы прост как выключатель, не нашлось.

### 2.2. Бизнес-цели

| Цель | Метрика | Приоритет |
|------|---------|-----------|
| Валидация продукта | Ежедневное личное использование 30+ дней | P0 |
| Первые пользователи | 100 установок | P1 |
| Органический рост | 500 установок без рекламы | P1 |
| Воронка в блог | 50 переходов в Telegram-канал OneZee | P2 |
| Retention | D7 > 30%, D30 > 15% | P1 |

### 2.3. Гипотеза

> Если дать пользователю инструмент, где чек-ин = 5 тапов без принятия решений, и показать прогресс красивой зелёной сеткой — он будет возвращаться каждый день, потому что: (а) чек-ин занимает 5 секунд, (б) разрыв серии визуально болезненный, (в) заполненная зелёная сетка = дофамин.

## 3. Целевые пользователи

**Персона 1: «Осознанный оптимизатор»** — 20–35 лет, хочет выстроить систему привычек. Пробовал сложные приложения — бросал. Готов потратить 5 секунд утром, но не 5 минут.

**Персона 2: «Контент-мейкер»** — 18–30 лет, ведёт блог о саморазвитии. Нужны красивые скриншоты прогресса.

**Анти-персона:** люди, ищущие КБЖУ-трекер, детальную аналитику, Apple Health синхронизацию.

## 5. Функциональные требования

### 5.1. FR-01: Бинарный чек-ин

**User Story:** Как пользователь, я хочу за 5 секунд отметить что я делал вчера, без оценок и размышлений.

**Acceptance Criteria:**

- Экран показывает вчерашнюю дату («15 февраля, воскресенье»)
- Список привычек — карточки-переключатели
- Тап по карточке: серая (—) ↔ зелёная (✓). Повторный тап — обратно
- Прогресс-бар внизу: заполняется по мере отметки (X/N)
- Кнопка «Готово ✓» → «День записан!» с саммари (N/M выполнено, чипы ✓/—)
- Шестерёнка (⚙) → настройки
- Дефолтное состояние: все карточки — «не делал» (—)

### 5.2. FR-02: Карточка-переключатель

**Механика:**

- Не тронута: белый фон, эмодзи, название, кружок с «—»
- Отмечена: зелёный фон (#E8F9ED light / rgba green dark), кружок зелёный с белой галочкой ✓
- Анимация: scale(0.97) при нажатии, spring-easing при отпускании
- Haptic feedback при каждом тапе
- Галочка появляется с pop-эффектом (scale bounce)
- Одного тапа достаточно — никаких подтверждений

### 5.3. FR-03: Прогресс — Годовой вид

- 12 карточек с мини-heatmap
- Бинарная цветовая система: зелёный (делал хоть что-то) / серый (ничего)
- В режиме «Все»: яркий зелёный (все привычки ✓), полупрозрачный зелёный (частично), серый (ничего)
- Процент выполнения на карточке месяца
- Сегодняшний день — пульсирующая зелёная рамка
- Легенда: Выполнено / Пропуск / Сегодня

### 5.4. FR-04: Прогресс — Месячный вид

- Календарная сетка, ячейки зелёные или серые
- Сегодня — пульсирующая рамка
- Текущая и лучшая серии
- Серия = дни подряд с хотя бы одной отмеченной привычкой

### 5.5. FR-05: Прогресс — Недельный / Дневной вид

- Неделя: карточки по привычкам, бары ✓/—, итог X/Y
- День: список привычек с ✓ или —, общий счёт

### 5.6. FR-06: Настройки

- Шестерёнка → bottom sheet
- Тёмная тема (toggle), соцсети (Telegram, Threads), версия

## 6. User Flows

### Ежедневный чек-ин (5 секунд)

```
Открыл → Тап ✓ → Тап ✓ → Тап ✓ → Тап ✓ → Тап ✓ → «Готово ✓» → Закрыл
```

### Drill-down прогресса

```
Прогресс → Год → Тап месяц → Месяц → Тап день → Неделя → Тап день → День
```

### Навигация

```
Tab Bar:
    ├── ◉ Чек-ин      + ⚙ настройки
    ├── ▦ Прогресс     год → месяц → неделя → день
    └── ⚙ Привычки    CRUD + reorder
```
